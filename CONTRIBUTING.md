# Contributing to MIT OCW Testing Suite

We welcome contributions to the MIT OCW Testing Suite project! Please follow these guidelines when contributing.

## How to Contribute

1.  **Fork the Repository:** Create a fork of this repository on GitHub.
2.  **Clone your Fork:** Clone your forked repository to your local machine.
    ```bash
    git clone https://github.com/noleysc/MIT-OCW-Testing-Suite.git
    ```
3.  **Set Up Development Environment:**
    *   Ensure you have Java Development Kit (JDK) and Maven installed.
    *   Run `mvn clean install` to build the project and download dependencies.
4.  **Create a New Branch:** Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/your-feature-name
    ```
    or
    ```bash
    git checkout -b fix/your-bug-fix
    ```
5.  **Make Your Changes:** Implement your feature or fix the bug.
6.  **Add Tests:** Ensure your changes are covered by new or existing tests.
7.  **Commit Your Changes:** Write clear and concise commit messages.
    ```bash
    git add .
    git commit -m "feat: Add new feature X"
    ```
    or
    ```bash
    git commit -m "fix: Resolve issue Y"
    ```
8.  **Push to Your Fork:** Push your branch to your forked repository.
    ```bash
    git push origin feature/your-feature-name
    ```
9.  **Open a Pull Request:** Submit a pull request from your branch to the `main` branch of the original repository.

## Code Style

Please adhere to the existing code style and conventions used in the project.

## Reporting Bugs

If you find a bug, please open an issue on the GitHub repository with a detailed description and steps to reproduce.

Thank you for contributing!
